Repository(Repo) Nedir?

Repo, Git klasörüne verilen isimdir. ".git" klasörünün bulunduğu klasörlerin hepsi birer repo olarak sayılmaktadır.

Git Init

Init, Git'i başlatmamızı sağlayan bir komuttur. Repo'muzu açıcağımız kısımda, komut satırında git init diyoruz ve repomuz yerel olarak açılmış oluyor.
Mevcut bir projeyi Git ile bir repo haline getirmek istiyorsanız, projenizin tüm klasörleri kapsadığı kısma gidin ve komut satırına git init komutunu girin. Artık projeniz Git ile bir repo haline gelmiş oldu...

Git Status

Status, "Giriş" sayfasında bahsettiğimiz dosya durumları bilgilerini çıktı olarak görebilirsiniz.
Örnek olarak bu dökümantasyonun reposunda benim bu sayfayı yazdığım sırada dosya durumlarını konsol üzerinden gözlemliyelim.

git status

Çıktı ise;
Gördüğünüz gibi istatislikler ile yaptığım işlemler ortada. Eğer daha kısa bir özet istersek;

git status -s

Çıktısı ise;
Daha kısa bir özet karşımda...

Git Add

Add, yapılan değişiklikleri Staged kısmına alma komutudur.
Commit atmadan önce pek tabi biz Git'e "Şu dosyalar güncellendi" diye bir uyarı vermeliyiz ki Git hangi dosyayı Repo'ya atıcağını bilesin.
Komutu kullanabilmek için; 
            
git add <eklenecekDosya>

Şeklinde kullanılır.

Tek Dosya ile Eklenme

Tek dosya ile eklemek için "eklenecekDosya" kısmına tek bir dosya ismi yazılır ve dosya Staged olmuş olur.
Testlerimizi gerçekleştirebilmek için bir repo açalım ve tüm testlerimizi bu kısımda gerçekleştirelim. İlk olarak bir klasör seçin ve git init komutunu yazıp çalıştıralım. Daha sonra içerisine bir merhabaGit.txt adlı bir dosya oluşturalım ve git add ile dosyamızı Staged yapalım.
git init

# Dosyayı oluşturalım

git add merhabaGit.txt

git status -s # Status ile de durumu kontrol edelim.

Çıktı ise;

Tüm Dosyaları ile Ekleme

Eğer birden fazla dosyayı Add ile eklicek isek "eklenecekDosya" kısmına "." koyarız. Bunun anlamı bulunduğu dizinin bütün dosyalarını ekliceğini belirtir.
Örnek olarak o test repomuza; "a.txt", "b.txt", "c.txt" ve "d.txt" adlı dört adet dosya oluşturalım ve ekliyelim.
        
# Dosyalar oluşturuldu

git status -s # İlk durumu görelim

git add .

git status -s # Son durumu gözlemliyelim


Çıktısı ise;
Gördüğünüz gibi tek bir komutla tüm dosyaları eklemiş olduk.

Belirli Uzantıları ile Eklemek

Sadece belirli bir uzantıya sahip dosyaları eklicek isek "eklenecekDosya" kısmına "*.istenenDosyaUzantisi" şeklinde yazmamız gerekir.
Örnek olması açısından, "main.c" ve "funcs.c" adında iki dosya; "main.cpp" ve "headers.cpp" adlı iki tane dosya oluşturalım. Ardından sadece ".cpp" uzantılı dosyaların eklenceğine dair bir add komutu girelim.
        
# Dosyaları oluşturalım

git status -s

git add *.cpp

git status -s

Çıktısı ise;
Böylelikle sadece belirli bir dosya uzantısını eklemiş olduk.

İnteraktif

Git, bizlere Add komutunu kullanmamız için özel bir TUI[Text User Interface(Yazısal Kullanıcı Arayüzü)] şeklinde bir ekran tasarlamışlardır. Bu ekrana giriş yapmak için git add -i diyerek girebiliriz. Ardından yönlendirmeler ile değişiklikler yapılabilir.
Git'in bize sunduğu nimetlerden yararlanmak için geri kalan o ".c" uzantılı dosyalarını da ekliyelim.

git add -i

# İşlemler...

git status -s

Çıktı ise;

Git Commit

Commit, yapılan değişiklileri kayıt etmek ve not eklemek için kullanılan bir komuttur. git commit -mod <atılacakOnaCommitMesajı>

Basit Commit

Basit bir Commit atmak için "mod" yerine "m" ve "atılacakOnaCommitMesajı" kısma ise çift tırnaklar içerisinde yazıcağımız yazabiliriz.
Örnek olsun niyetiyle klasikleşmiş bir Commit mesajı olan "Inital Commit" yani "Başlangıç Commit'i" mesajını yazalım.

git status -s # Önceki durumu görelim

git commit -m "Initial Commit"

git status -s # Son durumu gözlemliyelim


Çıktı ise;
Sondaki çıktı ise tüm dosyaların Commit'lendiğini ifade eder.

Editörlü Commit Atımı

Editör ile Commit atmak istersek sadece git commit yazmamız yeterlidir. Bizim başta tanımladığımız editörde Commit dosyasın açar, ardından kaydet ve çıkış yap deriz ve Commit tamamlanmış olur.
Test etmek için "e.txt" isimli bir dosya oluşturalım ki editörlü Commit'i deniyebilelim.

touch e.txt # Bash betik dilinde touch dosya oluşturmak için kullanılır. Touch ile dosyamızı oluşturuyoruz

git add e.txt

git status -s # İlk durumu gözlemliyelim

git commit # Gelen ekranda yazılacak olan Commit mesajı yazmak için, A tuşuna basalım. Bu bizi yazma modunu kullanmamızı olanak tanır. Ardından Commit mesajını yazalım. İşimiz bittiğinde ESC basalım ve gelen kısımda ":wq" yazalım ve işlem tamamdır

git status -s # Son durumu gözlemleyelim


Çıktıları ise;

İyi Commit Mesajı Yazma Prensibleri

İyi Commit yazmak, özellikle büyük projelerde yaptığınız değişiklikleri anlaşılması ve sürüm karmaşasını önliyerek hem anlaşılır repolar hemde sizlere bir prestij kazandırır. Gelin bakalım şu prensiblere;
İsmi Açıklama
Her Zaman İngilizce Biliyorum... Ana dilinizde yazmak istiyorsunuz fakat Dünya'nın genel olarak konuştuğu dil İngilizcedir. Evet yerel repolarınızda Türkçe kesinlikle kullanabilirsiniz fakat herkese sunduğunuz bir reponuz varsa, İngilizce Commit atmak her zaman bir tık daha öteye taşır.
Netlik Commit mesajınız her zaman açık ve net olmalıdır. Uzun uzun nedenler ve çözüm süreçleri yerine, "Şu hata şu çözüm ile giderildi" gibi bir mesaj daha etkili olucaktır. İlla hata olmasına gerek yok tabi, "Şu özellik eklendi" gibi net ve kesin cümlelerle Commit mesajları yazılmalıdır.
Göve Kısmını Açıklayıcı İfadeler Kullanılmalı Commit mesajlarının üç ana kısımları vardır. Bunlardan birisi az önce netlik başlığında bahsettiğimiz feat kısmı ile ilgiliydi. Açıklama yapılması gereken kısımlar işte fix kısmıdır. Bu kısımda sorunun neden çıktığı, nelere yol açıtğı ve nasıl düzeltildiği gibi soruların cevaplancağı bir açıklama metni yazılmalıdır. Docs kısmına ise kaynakça eklenmelidir.
Eki Olmayan Mesajlar Commit mesajı yazılırken bir diğer prensib ise emir kipi kullanımıdır. "Yapılmıştı" ifadesi yerine "Yapıldı" gibi kullanımlar daha çok tavsiye edilir.
Issues ve Pulrequests ile Bağlantılı Mesaj Yazma İlerleyen sayfalarda göreciğiz, ama Issues'lere Pulrequests'lere bağlantılı şekilde Commit mesajı yazmak, Commit mesajı karmaşasını engeller.
.gitignore Dosyası Kullanın İlerki sayfalarda daha detaylı işleceğiz. ".gitignore" bizler için aslında Git reposunda bulunan fakat sunucuya gitmesini istemediğimiz dosyaları veya klasörleri belirtmemizi sağlayan özel bir talimat listesidir. Genellikle kendi kendine oluşan veya derleme dosyayaları gibi gereksiz dosyalar ".gitignore" dosyasına yazılır. 
Bu prensiblere uyduğunuz sürece daha kaliteli Commit mesajları yazabilir hala gelebilirisiniz.

Formatlı Commit Mesajları

Formatlı Commit mesajları, Conventional Commits adlı bir kurum tarafından tüm Commit mesajlarının bir düzeni olması için ortaya atılmış bir formatlamadır.
Sıkça kullanılan bu formatlı Commit mesajlarındaki kullanılan kısımları tablo şeklinde inceleyelim.
İsmi Açıklama Örnek
feat: Yeni eklenen özellikler için kullanılır. Altına ise açıklama yapılabilir. feat: Bilim insanlarının Sözlerini Ekleme Özelliği Eklendi
fix: Düzeltilen bir hata için kullanılır. Altına açıklaması yazılabilir fix: CSS Görünümü Yenilendi
docs: Kaynakça olarak kullanılan bağlantılar eklenebilir. docs: https://data35delete.github.io/Data-s-Book-on-Git/

Git Log

Geçmişi görmek için kullanılan bir komuttur.
Direkt kullanımı size tüm Commit'lerde yapılan değişiklikleri tek tek detalı bir biçimde konsola yazdırıcaktır. Kullanmak için ise git log ile kullanılır.
Deneme amaçlı olarak tesk klasörümüzde;

git log

Komutunu çalıştıralım ve çıktıyı gözlemleyelim.
Özetini istersekde "--oneline" parametresini kullanabiliriz.

git log --oneline

Çıktı;
Dal şekilde istersekde "--graph" parametresini ekliyebiliriz.

git log --graph

Çıktı ise;
"-p" parametresi değişiklikleri görmemizi sağlarken, "--stat" ise tam teşekkürlü bir istatislik sunar bizlere.

Git Diff

Dosyalarda yapılan değişiklikleri gösteren bir komuttur.
Deneme alanımıza dönelim, main.c dosyasını içeriğini şu şekilde değiştirelim;

#include <stdio.h>

int main(){
    printf("Hello Git");

    return 0;
}

main.cpp dosyasını ise;

#include <iostream>

using namespace std;

int main(){
    cout << "Hello Git" << endl;

    return 0;
}


ardından

git add .

git diff --staged # Add ile alınan dosyalarda hangi değişiklikler yapıldığını gözlemleyelim

git commit -m "Kodlar eklendi"


Çıktısı ise;

, 
git show
